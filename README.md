# kcd-sponsor-engagement

Public web mirror of the **KCD Sponsor → Business Development engagement plan** for synvert xgeeks.
Live at: https://fabiosampaio-91.github.io/kcd-sponsor-engagement/

> **This repo is public and Google-indexable.** It holds published web assets and aggregated,
> role-based analysis only — **never** signed-contract contents, signatory/contact PII, sponsor
> pricing, internal cost numbers, or meeting notes. See [`CLAUDE.md`](CLAUDE.md).

The sensitive source-of-truth — signatory names, route-in contacts, per-deal sponsorship values —
lives only in `private/PRIVATE-account-playbook.md` (git-ignored) and the private companion repo.

## What this is

synvert xgeeks organises KCD Porto. This plan turns the resulting sponsor relationships into
nearshore-engineering pipeline for xgeeks — via a **dual-track** motion (sponsorship Role A +
business-dev Role B) that never compromises the sponsorship or KCD's vendor-neutral integrity.
Anchored to **KCD Porto 2026 (Nov 19–20, SuperBock Arena, Porto)**.

## Updating

1. Edit content here (or in the private companion repo's source).
2. Check the diff for any private/PII/pricing content (see warning above and `CLAUDE.md`).
3. Commit &amp; push — GitHub Pages redeploys automatically from `main`.

## Public vs. private — the rule

Anything pushed here may be cached or indexed even after deletion. The published site names
**companies and roles**, aggregates numbers, and keeps every individual's contact details,
every signed contract, and every per-deal value out. Hard, sensitive data stays private.
